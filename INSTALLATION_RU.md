# Установка и настройка бота профориентации (VK) — для начинающих

Здесь по шагам: что скачать, куда нажать, какие строки вписать в `.env`. Бот работает в **сообществе ВКонтакте**, анкеты и результаты дописываются в **Google Таблицу**, рекомендации строит **Ollama Cloud** по API.

Если что-то не получается — проверь раздел [Частые проблемы](#частые-проблемы) в конце.

---

## Что понадобится заранее

| Что | Зачем |
|-----|--------|
| Аккаунт VK и своё **сообщество** (группа) | Бот пишет от имени сообщества |
| Аккаунт **Ollama** (облако) | Генерация текста рекомендаций |
| **Google** (аккаунт + таблица + сервисный ключ) | Запись строк в Google Таблицу |
| **Python 3.11+** на компьютере или хостинг **Pterodactyl** с яйцом Python | Запуск кода |

---

## Шаг 1. Скачать проект и установить зависимости

### Windows (компьютер)

1. Скачай архив с GitHub (**Code → Download ZIP**) или клонируй репозиторий.
2. Распакуй в папку, например `C:\Users\Имя\career_bot_pterodactyl`.
3. Открой **PowerShell** или **cmd** в этой папке:
   - Shift + ПКМ по папке → «Открыть окно PowerShell здесь».
4. Создай виртуальное окружение и установи пакеты:

```powershell
python -m venv venv
.\venv\Scripts\activate
pip install -r requirements.txt
```

Если команда `python` не находится, попробуй `py -3.11 -m venv venv` и дальше снова `.\venv\Scripts\activate`.

---

## Шаг 2. Файл настроек `.env`

1. В папке проекта найди файл **`.env.example`**.
2. Скопируй его и переименуй копию в **`.env`** (именно так, без `.txt` в конце).
   - В проводнике Windows: если не видишь расширения, включи «Показать расширения имён файлов» в параметрах папки.
3. Открой `.env` в блокноте и заполни значения (ниже по разделам).

Пустые строки не трогай только там, где написано «необязательно».

## Шаг 3. ВКонтакте — сообщество и ключ

1. Зайди в [VK](https://vk.com) → **Сообщества** → создай группу или открой свою как администратор.
2. **Управление** → **Сообщения** → включи **Сообщения сообщества** (чтобы пользователи могли писать боту).
3. **Управление** → **Работа с API**:
   - Нажми создать **ключ доступа**.
   - Отметь права: доступ к **сообщениям сообщества** и разрешение **Long Poll API** (формулировки могут слегка отличаться — смысл тот же).
   - Скопируй длинный ключ — это **`VK_GROUP_TOKEN`** в `.env`.
4. **ID группы** — число из ссылки на группу:
   - Если ссылка вида `https://vk.com/club123456789`, то ID = **123456789**.
   - Если `public123456789`, то тоже **123456789**.
   - Впиши в **`VK_GROUP_ID`** только цифры, без `club` и `public`.
5. **Long Poll — типы событий** (в тех же настройках API / Long Poll):
   - Включи **входящее сообщение**.
   - Включи **действие с сообщением** (нужно для нажатий на кнопки в чате).

Сохрани `.env`.

## Шаг 4. Ollama Cloud

1. Зарегистрируйся на [ollama.com](https://ollama.com).
2. Зайди в настройки аккаунта и создай **API key**: [ollama.com/settings/keys](https://ollama.com/settings/keys).
3. Вставь ключ в `.env` в строку **`OLLAMA_API_KEY=`**.
4. **`OLLAMA_MODEL`** — модель, которая доступна твоему аккаунту (по умолчанию в примере указана одна из распространённых; при ошибке «model not found» смени на ту, что видишь в интерфейсе Ollama).
5. **`OLLAMA_BASE_URL`** обычно оставляй `https://ollama.com`, если документация не говорит иное.

## Шаг 5. Google Таблицы и сервисный аккаунт

Без **`GOOGLE_SPREADSHEET_ID`** и файла ключей бот всё равно отвечает в VK, но строки в таблицу не пишутся (остаётся локальный `results.json` на сервере).

### 5.1. Таблица

1. Создай новую [Google Таблицу](https://sheets.google.com).
2. Скопируй **ID** из адреса браузера.  
   Пример: `https://docs.google.com/spreadsheets/d/ABC123xyz/edit` → ID = **`ABC123xyz`**.  
   Впиши его в `.env` как **`GOOGLE_SPREADSHEET_ID=ABC123xyz`**.

### 5.2. Сервисный аккаунт (ключ JSON)

1. Открой [Google Cloud Console](https://console.cloud.google.com/) → создай проект или выбери существующий.
2. **APIs & Services → Library** — включи **Google Sheets API**.
3. **APIs & Services → Credentials → Create credentials → Service account** — создай сервисный аккаунт.
4. У созданного аккаунта: **Keys → Add key → JSON** — скачается файл, например `something-123456.json`.
5. Переименуй его в **`credentials.json`** и положи в **корень проекта** (рядом с `app.py`), **или** оставь любое имя и укажи полный путь в **`GOOGLE_APPLICATION_CREDENTIALS=`** в `.env`.

### 5.3. Доступ к таблице

1. Открой скачанный JSON в блокноте, найди поле **`client_email`** (вида `bot@....iam.gserviceaccount.com`).
2. В Google Таблице: **Настройки доступа** → добавь этот **email как редактора** (или «читатель» недостаточно — нужно право **редактора**, чтобы дописывать строки).

### 5.4. Лист

- По умолчанию бот пишет на **первый лист** книги.
- Если нужен конкретный лист, в `.env` задай **`GOOGLE_SHEET_WORKSHEET=ИмяЛиста`** (как внизу вкладки в таблице).

При первой записи, если лист **пустой**, бот сам добавит строку с **русскими заголовками** столбцов, затем — данные.

## Шаг 6. Запуск на своём компьютере

В папке проекта, с активированным `venv`:

```powershell
python main.py
```

или

```powershell
python app.py
```

Окно не закрывай — пока программа запущена, бот онлайн. Остановить: **Ctrl+C**.

Напиши своему сообществу в личку от другого аккаунта и проверь, что бот отвечает.

## Шаг 7. Pterodactyl (панель, яйцо Python)

Ниже — как это стыкуется с типовым **Startup Command** яйца Python (клонирование из Git, `pip install`, запуск `app.py`).

### Что делает стартовая команда (по шагам)

Твоя команда по сути делает следующее:

1. **`cd /home/container`** — рабочая папка сервера.
2. Если **нет папки `.git`** и задан **`GIT_ADDRESS`** — она **клонирует** репозиторий в текущую папку (все файлы проекта оказываются в `/home/container`).
3. Если **`AUTO_UPDATE=1`** — при каждом старте выполняется **`git pull`** (подтягиваются обновления с GitHub).
4. Если задан **`PY_PACKAGES`** — дополнительно ставятся указанные пакеты.
5. Если есть **`requirements.txt`** (имя из переменной **`REQUIREMENTS_FILE`**) — выполняется **`pip install -r ...`**.
6. В конце запускается **`python /home/container/${PY_FILE}`** — то есть файл, который ты указал в **App py file** (у тебя это **`app.py`**).

Итог: в корне контейнера должны лежать **`app.py`**, **`bot.py`**, остальной код и **`requirements.txt`** после клона; Python запускает **`app.py`**.

### Переменные панели — что выставить

| Переменная | Рекомендуемое значение |
|------------|-------------------------|
| **Git Repo Address** | `https://github.com/ProBRo141/career_bot_pterodactyl.git` (или твой форк; для приватного репо см. ниже) |
| **Git Branch** | Оставь пустым, чтобы брался branch по умолчанию (`main`), или укажи ветку явно |
| **User Uploaded Files** | **`0`** — чтобы при старте выполнялось клонирование/установка из Git (если поставить `1`, шаги с git часто пропускаются — тогда ты сам заливаешь файлы) |
| **Auto Update** | **`1`** — если хочешь автоматический `git pull` при рестарте; **`0`** — фиксированная версия с момента клона |
| **App py file** | **`app.py`** |
| **Additional Python packages** | Обычно **пусто** — всё уже в `requirements.txt` |
| **Requirements file** | **`requirements.txt`** |
| **Git Username** / **Git Access Token** | Нужны только если репозиторий **приватный**: тогда в `GIT_ADDRESS` подставляется URL с логином и токеном (как описано в подсказке панели) |

**Docker Image:** выбери **Python 3.11** (как у тебя в панели).

### Файлы, которые панель не возьмёт из Git (их нужно добавить вручную)

1. **`.env`** — в файловом менеджере сервера создай в **`/home/container/.env`** и вставь содержимое из `.env.example`, заполнив токены (VK, Ollama, Google и т.д.).  
   Код бота при старте ищет `.env` в `/home/container` — это как раз корень контейнера.

2. **`credentials.json`** — ключ сервисного аккаунта Google **нельзя** класть в публичный Git. Загрузи файл через **File Manager** в **`/home/container/credentials.json`**.  
   В `.env` должно быть: **`GOOGLE_APPLICATION_CREDENTIALS=credentials.json`** (путь относительно `/home/container`).

После этого нажми **Start** и смотри **Console** на ошибки (например, нет `.env`, неверный `VK_GROUP_ID`, таблица не расшарена на `client_email`).

## Переменные `.env` (шпаргалка)

| Переменная | Обязательно | Описание |
|------------|-------------|----------|
| `VK_GROUP_TOKEN` | Да | Ключ сообщества из настроек VK |
| `VK_GROUP_ID` | Нет* | Числовой ID группы (для vkbottle 4.8+ в коде не используется, можно оставить для заметок) |
| `OLLAMA_API_KEY` | Да для облака | Ключ с ollama.com |
| `OLLAMA_BASE_URL` | Нет | По умолчанию `https://ollama.com` |
| `OLLAMA_MODEL` | Нет | Имя модели |
| `GOOGLE_SPREADSHEET_ID` | Нет* | ID таблицы из URL Google Sheets |
| `GOOGLE_APPLICATION_CREDENTIALS` | Нет* | Путь к JSON, по умолчанию `credentials.json` в корне |
| `GOOGLE_SHEET_WORKSHEET` | Нет | Имя листа; если пусто — первый лист |
| `PROXY` | Нет | Если нужен прокси для исходящих запросов |

\* Без Google-переменных и файла ключей бот работает, но в таблицу не пишет.

Дополнительно в `.env` понимаются **алиасы Google**: `GOOGLE_SHEET_ID` вместо `GOOGLE_SPREADSHEET_ID`, `GOOGLE_CREDENTIALS_FILE` вместо `GOOGLE_APPLICATION_CREDENTIALS`.

## Частые проблемы

**Бот не отвечает в VK**  
Проверь: сообщения сообщества включены; ключ создан с нужными правами; в Long Poll включены **входящее сообщение** и **действие с сообщением**; `VK_GROUP_ID` — только цифры; бот на сервере реально запущен (логи без ошибок).

**Ошибка Ollama / 401 / model**  
Проверь `OLLAMA_API_KEY`, регион/ограничения аккаунта, имя модели в `OLLAMA_MODEL`.

**Google Sheets / Permission denied / API not enabled**  
Включён ли **Google Sheets API** в том же Google Cloud проекте, откуда JSON? Таблица расшарена на **`client_email`** из JSON с правом **редактора**? Верный ли **`GOOGLE_SPREADSHEET_ID`**?

**На Pterodactyl сразу падает**  
Проверь: **`User Uploaded Files = 0`**, если хочешь клон из Git; **`App py file`** = `app.py`; есть ли **`/home/container/.env`**; для Google — **`credentials.json`** в `/home/container`.
